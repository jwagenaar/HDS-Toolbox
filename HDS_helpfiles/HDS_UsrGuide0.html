Class definitions in Matlab


For a thorough explanation of classes and other Object Oriented programming, see Matlab's help files. In this section, only some of the basics are covered to understand the basic functionality necessary for using the HDS Toolbox.

Object Oriented programming handles data in the form of objects where each object has certain properties and methods. When we, for example, define an object named: "Car", we can give it certain properties such as: "Color", "Transmission type" and "Fuel type". Methods we can assign are: "Washing", "Driving" and unfortunately "towing"...

Class definitions are files that Matlab uses to define objects using exactly the same definitions: properties and methods. It is possible to have one object be a specific kind of another object (ie. "Electric cars" are a special type of "Cars") with additional properties ("Battery type", "Charge Time") and methods ("Charging"). Similarly, Matlab classes can be a subclass of another Matlab class and when we define a custom HDS class to store data, we make it a subclass of the HDS_main class.

In Matlab, class definitions are stored in folders on the Matlab path. Each class definition should be stored in a folder with the same name as the name of the object preset with an" @"-symbol. The filename of the class definition should also be the same as the object name. So, if we want to define an object with the name "Car", the class definition file should be:

.../SomeFolder/@Car/Car.m

where "SomeFolder" should be on the Matlab path.
