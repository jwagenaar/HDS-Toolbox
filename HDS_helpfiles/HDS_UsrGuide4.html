Dynamic properties

One of the core ideas behind the HDS Toolbox is that an object does not contain unnecessary information. Instead of preallocating all possible properties in an object, the user can add properties as needed. For example, during an experiment trial, we can record kinematics, force data, local field potential and many other data streams. However, if we would include those in all trial objects, each object would be cluttered with property names that are not used as we might only record a subset of the possible options. With the HDS Toolbox, the user only has to define in the class definition which object can be added but the user is not required to add all possible object types. This keeps the objects clean and managable.

There are two main methods to dynamically add properties to an object; Addobj and Addprop which are both discussed below. The last section of this page discussed 'Transient' objects/properties which are a special case of the previously mentioned option.

Adding and removing HDS Objects

Adding a HDS object to another HDS object will enable the user to index from one to the other using the '.'-notation. The user has to specify in the class-definition which classes can be added to an object of a particular class. (obj.dynamicVars)
